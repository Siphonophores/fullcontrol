## Library Customisations

### 1. Explicit retraction in `ender_3.py`
**File:** `fullcontrol/devices/community/singletool/ender_3.py`

The default `printer_command_list` inherited from `base_settings.py` uses
`G10`/`G11` (firmware retract), which requires the `FWRETRACT` feature
compiled into Marlin. Confirmed unsupported on this board via `M207` query.

**Fix:** override `retract` and `unretract` with explicit extrusion moves:
```python
"retract":   "G1 E-2 F2700 ; retract",
"unretract": "G1 E2 F2700 ; unretract",
```

### 2. Disabled stationary purge blob in `ender_3.py`
The original start sequence moved to X5 Y5 Z10, pushed 50mm³ of filament
as a stationary blob, lifted Z to 50mm to break the blob off, then travelled
to the start position. This is superseded by the `left_lines` primer (see
below) which primes more effectively against the bed. The four steps have
been commented out with a rationale note — re-enable if switching primers.

### 3. New primer: `left_lines.py`
**File:** `fullcontrol/gcode/primer_library/left_lines.py`

Two purge lines along the left edge of the bed (X≈0, Y20→Y200 and back),
matching the stock Cura/Ender 3 start script. Avoids the front-of-bed area
where physical clamps are mounted. Includes a retract/unretract around the
long diagonal travel to the print start point to suppress ooze.

Usage in notebook:
```python
initialization_data={'primer': 'left_lines', ...}
```

### 4. Shorter gcode filename in `steps2gcode.py`
**File:** `fullcontrol/gcode/steps2gcode.py`

Old format: `ripples__22-06-2026__16-36-26.gcode` (too wide for Ender 3 screen)
New format: `ripples_23jun_16-18.gcode`

```python
# before
datetime.now().strftime("__%d-%m-%Y__%H-%M-%S.gcode")
# after
datetime.now().strftime("_%d%b_%H-%M.gcode").lower()
```

### Brim (new cell 5 in notebook)
A radial brim is prepended to the main print steps. Since the shape is
defined in polar coordinates, adding `EW * loop` to the radius is equivalent
to Illustrator's "offset path" — exact perpendicular offset with no
approximation needed.

- Loops go **outermost to innermost** for a clean continuous connection into
  the first layer of the main print (no travel gap, no ooze risk)
- Uses the same ripple + star formula as the first layer (bulge term omitted
  — `sin(0) = 0` at z=0 anyway)
- No skew on brim loops
- Controlled by `brim_loops` variable (default: 3)

```python
for loop in range(brim_loops, 0, -1):  # outermost to innermost
    offset = EW * loop
    for t in range(int(layer_segs)):
        t_val = t / layer_segs
        a_now = t_val * tau - tau/4
        r_now = inner_rad + offset + \
            rip_depth * (...) + (tip_length * (...))
        brim_steps.append(fc.polar_to_point(brim_centre, r_now, a_now))
steps = brim_steps + steps
```

This approach works for any parameter combination in this notebook. It does
not generalise to non-polar shapes (those require a Clipper-style geometric
offset).
